сделать так, чтобы бэкенд управлял VPN-серверами и считал трафик.

Технологии: Xray-core (Linux), gRPC, Protobuf, bash/ansible.

3. Учет трафика (Stats Service):

- Сгенерировать Java-классы из proto-файлов Xray.
- Написать Job (Quartz/Spring Scheduler), который раз в 1-5 минут опрашивает ноды по gRPC, получает байты In/Out по каждому UUID (пользователю) и пишет в БД. Отключает UUID при нулевом балансе.

# ФАЗА 3: Mobile Network Core и Smart Mode

Реализация "Умного режима".

Технологии: Нативный Swift/Kotlin, Local DNS Proxy, IP Routing Tables.

Запихать 100 000+ заблокированных IP-адресов в таблицу маршрутизации OS (Split Tunneling по IP) — это смерть перфоманса. Делать нужно через DNS.

1. Реализация Fake DNS (Внутри мобилки):
- Настроить VPN так, чтобы он перехватывал все DNS-запросы телефона.
- Сделать локальный резолвер: если домен есть в списке blocked_domains (который приложение скачивает с бэкенда), возвращать фейковый IP (например, из подсети 198.18.0.0/15) и мапить его на настоящий домен внутри памяти. 
2. Умный роутинг (Tun2Socks / Xray Routing):
- Завернуть в TUN-интерфейс только трафик, идущий на фейковые IP-адреса.
- Внутри экстеншена переводить этот IP обратно в домен и отдавать Xray для туннелирования.
- Весь остальной трафик ОС (банки, Госуслуги) пускать напрямую мимо VPN (Bypass).
3. Реализация Connection Manager:
- Auto-reconnect при смене сети (Wi-Fi <-> LTE).
- Поддержание Keep-Alive.
